Overview

Purpose

The purpose of this site is to showcase the consulting team's software development capabilities, demonstrate past success, and generate high-intent leads. By offering a direct portfolio landing page and an interactive project cost estimator, potential clients can immediately evaluate the agency's work, calculate their potential investment, and easily request a consultation.

Audience

The target audience consists of founders, startups, small-to-medium business owners, and product managers who are looking for professional, high-quality custom software development services. They value transparent pricing, proven experience, and clean design, and they want a friction-free way to inquire about development estimates.

Dynamic elements

The website will feature the following dynamic JavaScript behaviors:

  • Portfolio Search & Filter: Real-time filtering of past projects by title, description keywords, and category tags ("All", "Web Apps", "Mobile Apps", "E-commerce") using Array.prototype.filter() to instantly update the DOM.
  • Project Detail Modal: Clicking any project card triggers a popup modal detailing the project's background, technologies, and client testimonials, powered by event listeners and dynamic DOM insertion.
  • Live Cost Estimator: A checkboxes form that updates the total estimated price dynamically on-screen as items are selected, powered by event listeners and pricing lookup objects.
  • Consultation Submission: Form validation for user contact info, which stores the customized project requirements and estimated costs into localStorage, displaying a confirmation success message upon submission.

Pseudo Code

Page 1: Portfolio Showcase (index.html)

  1. Define an array of objects ( const projects = [...]) containing mock portfolio project data (id, title, category, image, description, tech stack, testimonial).
  2. Implement a function displayProjects(projectList) that maps over the array, constructs template literals for each card, and injects them into the DOM project-grid container.
  3. Implement a function filterProjects(category) that triggers on filter button clicks, filters the project array using projects.filter(), and updates the display grid.
  4. Implement a function searchProjects(query) that triggers on search input change/keyup, filters matches in title or description, and re-renders the grid.
  5. Implement functions openModal(projectId) and closeModal() to toggle the modal overlay and dynamically load full details of the clicked project from the projects array.

Page 2: Interactive Estimator (estimator.html)

  1. Define an object containing the prices of services (const prices = { uiux: 1500, auth: 1000, ecommerce: 2000, dashboard: 2500, mobile: 3500 }).
  2. Implement a function calculateTotal() that gathers all checked feature inputs, retrieves their cost values, sums them, and updates the total estimate display panel.
  3. Add change event listeners to recalculate the estimate instantly whenever features are toggled.
  4. Implement a function submitConsultation(event) that validates inputs (Name, Email), retrieves the calculated estimate and selected items, saves this data into localStorage, and displays a success notification.
